# Contributing to Unbound Intelligence

First off, thank you for considering contributing! It’s the "wild west" tinkerers and "Big Iron" enthusiasts who build the most resilient systems.

### How Can I Contribute?

#### Reporting Bugs
* Check the GitHub Issues tab to see if the bug has already been reported.
* If not, open a new issue. Include your Python version, the input text that failed, and the expected versus actual result.

#### Suggesting Enhancements
* We are always looking for better Regex patterns for refusal detection.
* If you have a suggestion, open an issue first to discuss the logic.

#### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add some test cases to the `if __name__ == "__main__":` block or a separate test file.
3. Ensure your code adheres to Python PEP 8 standards.
4. Write a clear, concise commit message.

### Our Philosophy
We value **deterministic reliability**. Every evaluator should provide a clear, repeatable "Pass" or "Fail" with a human-readable reason. We prioritize logic that can run on anything from a decentralized cluster to an IBM Mainframe.
